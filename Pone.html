Plagiarism Detector using AWS

The Streamlit app is designed to detect plagiarism and find similarities between texts integrated with AWS S3 and EC2. It allows users to enter text or upload files (docx, pdf, txt). It splits the text into sentences, searches Google for each sentence, and compares the input text with retrieved results using cosine similarity. Users can also compare the similarity between two uploaded files. The results are displayed in a formatted DataFrame.

Building a Plagiarism Detection Tool with Streamlit and AWS S3

In the age of digital content, ensuring originality is crucial. Whether you're a student, researcher, or content creator, plagiarism detection tools can help maintain integrity. In this article, we will walk through the development of a Plagiarism Detector using Streamlit, Natural Language Processing (NLP) techniques, and AWS S3 for file storage.

Key Features

Our plagiarism detection tool offers:

  • Text Input or File Upload: Users can manually enter text or upload a .docx, .pdf, or .txt file.
  • Sentence Tokenization: Text is broken into sentences for comparison.
  • Web Search for Similar Content: The tool fetches URLs from Google search results.
  • Cosine Similarity Analysis: Detects similarity between uploaded content and online sources.
  • AWS S3 Integration: Securely uploads and downloads files from an S3 bucket.

Original Key Features

  • User Input Options:
    • Enter text directly into a text area.
    • Upload a single file in .docx, .pdf, or .txt format.
    • Upload two files for similarity checks.
  • Text Processing: The application can read text from different file formats (.txt, .pdf, and .docx).
  • Plagiarism Detection: For entered text or uploaded files, the application searches Google for each sentence. It retrieves the first URL from the search results and checks the similarity of the text at that URL with the input text.
  • Similarity Check between multiple files: When users upload files, the application calculates the similarity between the texts of the files. It uses cosine similarity to measure how similar the texts are.

To integrate AWS S3 for secure file storage, first set up an S3 bucket on AWS and create IAM user credentials.

Steps to Set Up AWS S3

Create an S3 Bucket:

  1. Go to the AWS Management Console.
  2. Navigate to S3 and click Create bucket.
  3. Choose a unique bucket name and select the region.
  4. Adjust bucket settings as needed and create the bucket.

Set Up IAM User Credentials:

  1. Navigate to AWS IAM (Identity and Access Management).
  2. Create a new user with programmatic access.
  3. Attach the policy AmazonS3FullAccess (or create a custom policy for restricted access).
  4. Save the Access Key ID and Secret Access Key securely (we will use these keys in the code).

Additional Notes

  • Security: Ensure your AWS credentials and S3 permissions are set correctly to avoid access issues.
  • Dependencies: Make sure all required libraries are installed and configured properly on your EC2 instance.
